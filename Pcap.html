Pcap File Analyser

About The Program
This was also written as part of a team, we had to sift through a pcap file without using Wireshark to find information in the file, such as what the user was accessing, what was the dodgy search engine, what endian the header was etc, this was quite a repetitive project as it mostly involved reading bytes from the file and converting them into something legible and present the data in such a way that the user was able to see all the relevant information.
How it Works

This was spilt into four tasks each with a different output required, so if the user only wanted to see the structure of a pcap file, they would only need to run the first part, and if they needed other information, they could run other parts of the program separately. All sections were written in python.

Section One (Global Header)

The Function of this section was to analyse the global packet header. The output for this section was therefore the length of the header, the magic number and the endianness of the pcap file ( this is important because it changed the order and how you read the file), the major and minor version numbers of the file format, the snap length and the data link type. This section was very important for the other sections because it determined how the rest of the file was going to be read and understood.

Section Two (DCHP Frame)

This section's function was the analyse the DHCP frame and provide the user with; the timestamp of when this packet was captured, the actual GMT time corresponding to the timestamp, the length of the DHCP frame, the source and destination MAC and IP addresses of the captured comunication, and finally the name of the host PC.

Section Three (Find The Dodgey Website)

Section three was failry easy to implement as all we had to do was find a domain name which has an abnormal ending, so for this we searched through the whole file looking for domain names, and saving them, if they had a .com etc they were saved to a "trusted" list and if the ended in anything not in the safe names we added them to a separate list, once the program had searched the whole file the abnormal list was displayed to the user.

Section Four (Search engine)

For this section, similar to the previous one, we used a list of known search engines and if the user accessed a popular search engine it was logged, this means we could see what site the user accessed and where they got the url from in the first place and what they searched for.
