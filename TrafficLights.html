Traffic Management System

About The Program
This program was written in C++. or more specifically the Arduino variant of the language, the Idea of this system was to create a completely embedded system which will run autonomously without any need for any input from outside users. Once the system has been installed it will operate completely by itself. I did this by adding sensors to the “traffic Light” which for a proof of concept was an Arduino breadboard with some LEDs. The sensors I added to the system were a distance sensor which would be used to detect if there was a car at the lights or if there was a pedestrian at the crossing depending on the day variable. This was controlled by the light dependant resistor which would tell the camera it if was day or night. This would then change the use of the distance sensor. If it were daytime and a pedestrian walked up to the crossing the lights would change from green to red. If it were in night mode the sensor would have the lights on red until a car approaching was detected, at which point it would change the lights from red to green to allow the car to pass.
How it Works

I kind of got carried away talking about this program in the above section because I vaguely described how it worked already. But briefly, there are some functions which change depending on sensor readings which change the light characteristics of the system. I.E. Day mode, Night Mode, Pedestrian centric mode, and car centric mode. This would allow the programmer to change the specific characteristics of the light changes depending on the esoteric application of one of the systems. If it were positioned in a more pedestrian area it would obviously be more biased towards the pedestrian mode and the lights would stay red for longer.
